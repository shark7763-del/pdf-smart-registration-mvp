# Phase 1 雲端連動 — 部署說明（照做即可）

做完這份，家長用手機填的報名，你的教練後台就會即時收到（資料存在你的 Google 試算表，附件存在你的 Google 雲端硬碟）。

> 同一個 `index.html` 有兩種模式：
> - **被 GAS 服務時**（用下面的 /exec 網址打開）→ 走雲端、多人共用。
> - **直接開檔 / GitHub Pages** → 自動變回單機展示 demo（不連動）。
> 所以部署後，要把 **/exec 網址** 發給家長，不是 github.io 的網址。

---

## 一次性部署（約 10 分鐘）

1. 用「**要放正式資料**」的 Google 帳號，到 [sheets.new](https://sheets.new) 新建一份 Google 試算表，命名例如「比賽報名資料」。
2. 在這份試算表上方：**擴充功能 → Apps Script**。
   （一定要從這份表進去，這樣程式才會綁定到它。）
3. 左側 `Code.gs`：把本資料夾 `Code.gs` 的內容**全部貼進去**取代。
4. 左側按 **＋ → HTML**，檔名打 **`Index`**（不要打 .html）；把 `index.html` 的內容**全部貼進去**。
5. 上方函式選單選 **`setupSheets`**，按 **▷ 執行**。第一次會跳授權 → 選你的帳號 → 「進階 → 前往（不安全）」→ 允許。
   （執行後回試算表會看到 Events / Registrations / Settings 三張表。）
6. 右上 **部署 → 新增部署 → 齒輪選「網頁應用程式」**：
   - 執行身分：**我**
   - 誰可以存取：**任何人**
   - 按「部署」，複製 **/exec 網址**。
7. 用手機開那個 /exec 網址測一筆報名 → 回試算表看 Registrations 有沒有出現。有 → 成功 🎉

把 /exec 網址發給家長即可。教練後台：同一個網址，右上「教練功能 → 教練後台」，密碼預設 **1234**（進後台可改）。

---

## 之後要改程式、重新部署（很重要，照做才不會出錯）

> 改了 `Code.gs` 或 `Index`，**/exec 不會自動更新**，要手動發新版本：

**部署 → 管理部署作業 → ✏️（編輯）→ 版本選「新版本」→ 部署。**

⚠️ 千萬**不要**按「新增部署」或另開新的 Apps Script 專案 —— 那會給你一個**新網址**、而且綁到**另一份空白試算表**，看起來像「資料全不見」（其實舊網址的資料還在）。

想快速測試又不想每次發版本：用「**測試部署作業**」的 `/dev` 網址（永遠跑最新存檔，但只有你本人開得了）。

---

## 常見狀況

- **右上連線狀態**：「已連雲端」=正常；「本機展示模式」=你開到的是 github.io 或本機檔（不連動）；「雲端連線失敗」=檢查網路或部署存取權是否設「任何人」。
- **看不到別人剛送的報名**：後台按「**重新整理**」即可拉最新（雲端不是即時推播，是按需重撈）。
- **附件**：家長上傳的照片/PDF 會存到你雲端硬碟的「**比賽報名附件**」資料夾，後台名單可直接點連結看。單檔上限 2MB。
- **改欄位**：`Code.gs` 的 `HEADERS` 新增欄位**一律加在陣列最後**，不可插中間（否則舊資料會整批讀錯欄）。
